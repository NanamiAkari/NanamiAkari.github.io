1. 什么是 Docker？

简单理解

Docker 就像一个超级轻量级的虚拟机，但比虚拟机更快、更省资源。

生活中的类比：

  • 传统部署 = 在不同的房子里装修（每个房子都要重新买家具）
  • Docker = 集装箱运输（标准化容器，到哪都能用）

技术定义

Docker 是一个容器化平台，它可以将应用程序及其依赖项打包到一个轻量级、可移植的容器中。

问题1：项目迁移后，环境不一致导致崩溃

Docker 的优势

回滚麻烦 秒级回滚

3. Docker 核心概念

三大核心组件

镜像 (Image)

  • 定义：应用程序的”模板”或”蓝图”
  • 类比：就像软件安装包（.exe 文件）
  • 特点：只读、分层存储、可复用

容器 (Container)

  • 定义：镜像的”运行实例”
  • 类比：就像运行中的程序进程
  • 特点：可读写、隔离、临时

仓库 (Repository)

  • 定义：存储镜像的地方
  • 类比：就像应用商店（App Store）
  • 主要仓库：Docker Hub（官方）、阿里云、腾讯云
